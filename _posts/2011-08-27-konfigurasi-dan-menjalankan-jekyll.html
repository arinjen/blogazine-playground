Menyambung dari tulisan sebelumnya yaitu, instalasi Jekyll di Ubuntu, saya melanjutkan untuk bagian konfigurasi awal.

Saya menggunakan Github untuk hostingnya, maka kita login terlebih dahulu di Github dan buat repositori baru dengan nama username.github.com. Untuk konfigurasi dan instalasi Git di komputer kamu, bisa dibaca di http://help.github.com. Di sana dijelaskan secara rinci bagaimana men-setup Git. Untuk panduan dengan bahasa Indonesia, saya menemukan tulisan Belajar Git dengan Github.

Setup Folder

Oke. Sekarang masuk ke folder home (saya pakai Ubuntu) dan buat folder dengan nama sama dengan nama repositori yang tadi dibuat, yaitu username.github.com. Masuk ke folder tersebut. Lalu buat struktur seperti di bawah ini: {% highlight bash %} . |-- .gitignore |-- README |-- _config.yml |-- _layouts | -- layout.html | -- post.html |-- _posts | -- 2011-08-27-judul-post.markdown |-- index.html {% endhighlight %}

Dan berikut adalah penjelasan tiap folder dan file di atas:

  1. .gitignore

    Ini bukanlah file penting yang dibutuhkan Jekyll. File ini berguna ketika kita akan meng-commit ke Github. Isi file ini adalah daftar file atau folder yang tidak akan di upload ke Github.

  2. README

    Ini juga bukan file yang penting. Tapi disarankan untuk setiap repositori Github. File ini berisi deskripsi singkat tentang repositori.

  3. _config.yml

    File ini berfungsi untuk menyimpan konfigurasi setting Jekyll. Jika file ini kosong, Jekyll akan menggunakan settingan default. Untuk daftar konfigurasi yang ada dan settingan default, bisa ditemukan di wiki: Configuration.

  4. _layouts

    Folder ini berfungsi untuk menyimpan semua template layout. Berikut adalah 2 contoh file layout:

    • default.html

      Ini adalah basic template kita. Tidak ada peraturan penamaan. Jika diganti, jangan lupa untuk mengganti nama layout pada YAML Front Matter blocks.

    • post.html

      Ini adalah basic template untuk post page. Untuk selanjutanya tentang YAML Front Matter blocks bisa dibaca di wiki: YAML Front Matter.

  5. _posts

    Ini adalah folder untuk postingan yang telah ditulis. Peraturan penamaan file dibutuhkan disini. Diawali dengan tanggal lalu judul post dengan susunan tahun-bulan-tanggal-judul-post.markdown. Contoh: 2011-08-27-judul-post.markdown. Nama file ini akan digunakan juga untuk permalink post.

    • 2011-08-27-judul-post.markdown

      Isi postingan ditulis di file ini. File ini ditulis menggunakan syntax markdown, cek juga dokumentasi syntax Markdown. Jangan lupa untuk memasukkan YAML Front Matter pada awal file: {% highlight bash %} --- layout: post title: Judul post --- {% endhighlight %}

  6. index.html

    File ini berisi halaman utama blog. Bisa diisi sesuai selera tergantung apa yang akan dimunculkan di front-page.

Saatnya pergi ke Terminal

Sekarang buka Terminal dan masuk ke direktori username.github.com. Masukkan perintah {% highlight bash %} jekyll --server --auto {% endhighlight %} Buka browser dan masuk ke http://localhost:4000. Lihat hasilnya.

Jekyll menggunakan Liquid Markup untuk templating. Daftar data template yang tersedia bisa dilihat di wiki: Template Data dan wiki: Liquid Extensions.
